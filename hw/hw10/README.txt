DISCUSSION

Overall, most of the order notation for each operation of all data structure match my analysis. I did update the remove duplicate of priority queue, i was assuming
that operation is feasible, and it turn out to be not. Other than that most operation matches.
During the execution of the program for each data structure, i find out that the priority queue has a dominant executing speed when dealing with larger number of string
other data structure are having trouble with the operation of longest substring and closest value when the number of string is 50000 or 100000.
Another observation is that the string normally takes longer execution time than integer, except for priority queue.
The coefficient for longest substring of hash and bst are similar, but are a little bit different.
For most operation, bst, vector , list are all similar, except hash table has a slightly faster execution time.
So after this hw, i think if i will work with large data, i will probably go with priority queue among others.
As for the memory usage i think they are all similar.
